# Caveman communication (always active)

Caveman policy applies to **orchestration chat** only. Written artifacts (brainstorm, PRD, tasks,
amendments, commit bodies, PR descriptions) use **normal complete prose** always.

## Intensity levels

| Level | Chat style |
|-------|------------|
| **normal** | Standard complete prose; caveman off |
| **lite** | Tight professional; articles kept |
| **full** | Classic caveman: drop articles, fragments OK, short synonyms |
| **ultra** | Max compression |

## Auto-Clarity

Write normally for security warnings, irreversible confirmations, commit/PR bodies, multi-step sequences
where fragments risk misread, or anywhere terseness creates technical ambiguity. Resume routed intensity after.

## Boundaries

Code blocks, commits, PR descriptions, and generated user-facing docs use normal complete prose regardless
of chat intensity.
